ES6 Modules: The diary

In its early days, Javascript programs were pretty small. Over time, splitting code into separate files became a necessity. Since 2015, ES6 has introduced modules natively to JavaScript.

Your goal: Display and create diary entries while splitting code with modules.

Level 1/2: Displaying Entries
  • You will have 3 JS files for this exercise:
    • index.js is your main module. It should only select elements from the DOM and set event listeners
    • diaryFns.js is where all the functions for your project will be described, and exported.
    • data.js will contain data to be read.
  • data.js is already provided, with a few additional diary entries. Analyze the structure of the data you were given.
  • Create index.js, and add it to your HTML as a module. It should select the section with the id of allDiaryEntries.
  • Create diaryFns.js, and inside:
    • Import the variable from data.js
    • Create a function that will loop over that data, create all needed elements for a new diary entry, and add the contents of each object in the array in the appropriate elements.
    • If unsure, have a look at the markup in index.html to see what kind of elements and properties you need for each diary entry card!
    • As the section where your new entry will be added is selected in another file, think about how you can send a reference of that node to the function you created in dairyFns.js (without selecting the element in diaryFns.js!)
    • Export this function, as a named export, from diaryFns.js, then import and call it in index.js

Level 2/2: Adding new Entries
  • In diaryFns.js, create and export (as a named export) a function that:
    • Creates elements for a new diary entry based on the content of the inputs in the form, and adds it to the section selected in index.js
    • ⚠️Existing elements should not be selected in diaryFns.js⚠️ index.js should call your exported function, and pass all nodes to that function!
  • In index.js, import the above function from diaryFns.js, and add it to an event listener that will trigger upon submit of the form. Remember to reset the form as well!
